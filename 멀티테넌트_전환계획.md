  /tenants/{tenantId}/settings/
  /tenants/{tenantId}/drivers/

## 알아둘 점
- **배포 후 캐시 문제**: GitHub Pages는 CDN/브라우저 캐시가 있어서, 새로고침만으로는 변경사항이 안 보일 수 있음. URL 끝에 `?v=2` 같은 캐시버스터를 붙이거나 시크릿창에서 확인 권장.
- ~~**⚠️ 서비스워커 캐시 (미해결)**~~ → **2026-06-23 해결 완료**: 네트워크 우선(network-first) 전략의 `sw.js`를 신규 작성·배포. 설치 즉시 모든 구 캐시 삭제 + `skipWaiting` + `clients.claim()`으로 열린 탭 즉시 제어권 획득. 이후 배포부터는 새로고침 한 번으로 최신 파일 반영됨.
- **OMS 내부 2차 로그인**: app.html 진입 후 별도로 뜨는 로그인은 테넌트별 직원 계정(`settings/omsAccounts`)이며, 최초 1회는 자동으로 admin/wellbing1234 계정이 생성됨. (2026-06-22부터 최초 로그인 시 비밀번호 변경 강제됨 — 위 5단계 참고)
- **같은 브라우저에서 관리자/기사 동시 로그인 주의**: app.html(관리자)과 driver.html(기사)이 같은 브라우저에서 동시에 열려 있으면, 한쪽에 로그인하면 다른 쪽이 로그아웃될 수 있음 (Firebase Auth 로그인 상태가 브라우저 단위로 공유되기 때문). 보통 관리자/기사가 다른 기기를 쓰므로 실사용에는 문제 없음.
